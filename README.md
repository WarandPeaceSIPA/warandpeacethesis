## 🚀 Quick Start

### Option 1: GitHub Web Interface (Easiest - 5 minutes)

1. Go to your GitHub repository
2. Click "Add file" → "Upload files"
3. Drag and drop all files from this folder
4. Write commit message: "Initial commit - Research website v3.6"
5. Click "Commit changes"
6. Wait 5 minutes for GitHub Pages to build
7. Visit: `https://[your-username].github.io/[repository-name]/`

### Option 2: Git Command Line

```bash
# Initialize repository
git init

# Add all files
git add .

# Commit
git commit -m "Initial commit - Research website v3.6"

# Add remote (replace with your repository URL)
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git

# Push to GitHub
git push -u origin main
```

### Option 3: Quick Deploy Script (Automated)

```bash
# Make script executable
chmod +x quick-deploy.sh

# Run the script
./quick-deploy.sh
```

## 🐛 Troubleshooting

### Website not loading?
- Wait 10 minutes after deployment
- Check GitHub Pages settings
- Clear browser cache (Ctrl+Shift+R)
- Try incognito mode

### Analytics not working?
- Verify you replaced placeholder IDs
- Wait 15 minutes after deployment
- Check browser Console (F12) for errors
- Ensure IDs are correct format

### Dashboard not loading?
- Verify `dashboard.html` is uploaded
- Check `dashboard-data.json` is present
- Clear browser cache
- Check Console for errors
