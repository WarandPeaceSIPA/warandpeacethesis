## <a name="step1"></a>الخطوة 1️⃣: إنشاء حساب GitHub

### إذا كان لديك حساب بالفعل، انتقل إلى [الخطوة 2](#step2)

### 1.1 الذهاب إلى موقع GitHub
- افتح المتصفح واذهب إلى: **https://github.com**
- انقر على زر **"Sign up"** في الزاوية العلوية اليمنى

### 1.2 ملء معلومات التسجيل
أدخل المعلومات التالية:
- **Email**: بريدك الإلكتروني (مثل: zmalessi@dah.edu.sa)
- **Password**: كلمة مرور قوية (على الأقل 15 حرف)
- **Username**: اسم المستخدم (مثل: zaher-alessi)
  - ⚠️ **مهم جداً**: احفظ اسم المستخدم هذا - ستحتاجه لاحقاً!

### 1.3 التحقق من الحساب
- حل اللغز البسيط (CAPTCHA)
- انقر **"Create account"**
- افتح بريدك الإلكتروني وانقر على رابط التأكيد

### 1.4 إكمال الإعداد
- اختر **"Free"** (مجاني)
- يمكنك تخطي الأسئلة الاختيارية
- انقر **"Complete setup"**

✅ **تم إنشاء حسابك بنجاح!**

## <a name="step2"></a>الخطوة 2️⃣: إنشاء Repository جديد

### 2.1 الذهاب إلى صفحة إنشاء Repository
- بعد تسجيل الدخول، انقر على زر **"+"** في الزاوية العلوية اليمنى
- اختر **"New repository"**

### 2.2 إعدادات الـ Repository

#### اسم الـ Repository (مهم جداً):
```
warandpeacethesis
```
⚠️ **انسخ هذا الاسم بالضبط - لا تغيره!**

#### الإعدادات الأخرى:
- **Description** (اختياري): 
  ```
  Master's Thesis: Informational Conditions for Structural Constraint Effectiveness
  ```
- **Public**: ✅ اختر هذا الخيار (إلزامي لـ GitHub Pages المجاني)
- **Add a README file**: ❌ لا تضع علامة (لدينا README جاهز)
- **Add .gitignore**: ❌ لا تختر (لدينا .gitignore جاهز)
- **Choose a license**: ❌ لا تختر

### 2.3 إنشاء الـ Repository
- انقر على الزر الأخضر **"Create repository"**

✅ **تم إنشاء الـ Repository بنجاح!**

## <a name="step3"></a>الخطوة 3️⃣: رفع ملفات الموقع

### 3.1 الذهاب إلى صفحة رفع الملفات
في صفحة الـ Repository الجديد، ستجد رسالة:
> "Quick setup — if you've done this kind of thing before"

أسفلها، انقر على الرابط: **"uploading an existing file"**

### 3.2 تحضير الملفات
افتح مجلد `warandpeacethesis-github` على جهازك، وستجد هذه الملفات:
```
✅ index.html
✅ script.js
✅ styles.css
✅ papers-database.json
✅ package.json
✅ logo.svg
✅ README.md
✅ .gitignore
✅ CNAME
✅ DEPLOYMENT_GUIDE.md
✅ QUICK_START.md
```

### 3.3 رفع الملفات

#### الطريقة الأولى: السحب والإفلات (الأسهل)
1. حدد **جميع الملفات** من المجلد
2. اسحبها وأفلتها في منطقة الرفع على صفحة GitHub
3. انتظر حتى يكتمل الرفع (سترى شريط التقدم)

#### الطريقة الثانية: اختيار الملفات
1. انقر على **"choose your files"**
2. حدد جميع الملفات من المجلد
3. انقر **"Open"**

### 3.4 إتمام الرفع
في أسفل الصفحة:
- **Commit message**: اترك الرسالة الافتراضية أو اكتب:
  ```
  Initial commit: War and Peace Thesis website
  ```
- انقر على الزر الأخضر **"Commit changes"**

⏳ **انتظر 10-20 ثانية حتى يكتمل الرفع...**

✅ **تم رفع الملفات بنجاح!**

## <a name="step4"></a>الخطوة 4️⃣: تفعيل GitHub Pages

### 4.1 الذهاب إلى الإعدادات
- في صفحة الـ Repository، انقر على **"Settings"** (أيقونة الترس)

### 4.2 الذهاب إلى صفحة Pages
- في القائمة الجانبية اليسرى، ابحث عن **"Pages"**
- انقر عليها

### 4.3 إعدادات النشر

#### Source (المصدر):
- في قسم **"Build and deployment"**
- تحت **"Source"**، اختر: **"Deploy from a branch"**

#### Branch (الفرع):
- في قائمة **"Branch"**، اختر: **"main"**
- في القائمة الثانية، اختر: **"/ (root)"**
- انقر على زر **"Save"** الأزرق

### 4.4 انتظار النشر
⏳ **انتظر 1-2 دقيقة...**

ستظهر رسالة في الأعلى:
> ✅ Your site is live at https://YOUR_USERNAME.github.io/warandpeacethesis

✅ **تم تفعيل GitHub Pages بنجاح!**

## <a name="step5"></a>الخطوة 5️⃣: الوصول إلى الموقع

### 5.1 الحصول على الرابط
رابط موقعك سيكون بهذا الشكل:
```
https://YOUR_USERNAME.github.io/warandpeacethesis
```

**مثال:**
- إذا كان اسم المستخدم: `zaher-alessi`
- الرابط: `https://zaher-alessi.github.io/warandpeacethesis`

### 5.2 زيارة الموقع
- انقر على الرابط الأخضر في صفحة Settings > Pages
- أو انسخ الرابط والصقه في المتصفح

### 5.3 التحقق من الموقع
تأكد من أن:
- ✅ الصفحة تفتح بشكل صحيح
- ✅ شعار جامعة دار الحكمة يظهر
- ✅ الألوان البنية والذهبية تظهر بشكل صحيح
- ✅ قاعدة البيانات تعمل
- ✅ الرسوم البيانية تظهر

🎉 **مبروك! موقعك الآن متاح على الإنترنت!**

## <a name="step6"></a>الخطوة 6️⃣: تحديث الموقع لاحقاً

### لتحديث أي ملف:

#### 6.1 الذهاب إلى الملف
- اذهب إلى صفحة الـ Repository الرئيسية
- انقر على الملف الذي تريد تعديله (مثل: `index.html`)

#### 6.2 تعديل الملف
- انقر على أيقونة القلم (Edit) في الزاوية العلوية اليمنى
- قم بالتعديلات المطلوبة

#### 6.3 حفظ التعديلات
- في أسفل الصفحة، اكتب وصف التعديل (مثل: "تحديث معلومات الاتصال")
- انقر **"Commit changes"**

⏳ **انتظر 1-2 دقيقة، ثم حدّث صفحة الموقع**

✅ **تم تحديث الموقع!**

## <a name="troubleshooting"></a>🔧 حل المشاكل الشائعة

### المشكلة 1: الموقع لا يفتح (404 Error)
**الحل:**
1. تأكد من أن الـ Repository اسمه بالضبط: `warandpeacethesis`
2. تأكد من تفعيل GitHub Pages في Settings > Pages
3. انتظر 5 دقائق إضافية (أحياناً يستغرق وقتاً)

### المشكلة 2: الألوان لا تظهر بشكل صحيح
**الحل:**
1. تأكد من رفع ملف `styles.css`
2. امسح ذاكرة التخزين المؤقت (Cache) في المتصفح: Ctrl+Shift+R

### المشكلة 3: الشعار لا يظهر
**الحل:**
1. تأكد من رفع ملف `logo.svg`
2. تحقق من أن اسم الملف صحيح (حساس لحالة الأحرف)

### المشكلة 4: قاعدة البيانات لا تعمل
**الحل:**
1. تأكد من رفع ملف `papers-database.json`
2. تأكد من رفع ملف `script.js`
3. افتح Console في المتصفح (F12) وتحقق من الأخطاء

### المشكلة 5: الرسوم البيانية لا تظهر
**الحل:**
1. تحقق من اتصالك بالإنترنت (Chart.js يُحمّل من CDN)
2. جرب متصفح آخر

## ✨ مميزات إضافية

### إضافة نطاق مخصص (Custom Domain):
إذا اشتريت نطاق مثل `warandpeacethesis.com`:
1. في Settings > Pages
2. أضف النطاق في "Custom domain"
3. اتبع تعليمات إعداد DNS من مزود النطاق

### تفعيل HTTPS:
- ✅ تلقائي! GitHub Pages يوفر HTTPS مجاناً
- تأكد من تفعيل "Enforce HTTPS" في Settings > Pages

### إضافة Google Analytics:
لتتبع زوار الموقع، أضف كود Google Analytics في ملف `index.html`
